# 测试函数

说明

下文以函数latest版本为例介绍测试方法。如果需要测试函数的已发布版本，可在已发布版本详情页面选择“函数代码”页签，参考方式二进行测试。

函数创建后可以在AGC控制台测试函数的代码运行是否正常。进入测试界面有两种方式：

-   方式一：函数列表中点击函数名称右侧“操作”列的“测试”，在右侧弹出的“测试函数”界面进行测试。
    
    ![](https://contentcenter-vali-drcn.dbankcdn.cn/pvt_2/DeveloperAlliance_scene_100_1/84/v3/CmSrIOjaQDC1CsLMfiqY0Q/zh-cn_image_0000002474214321.png?HW-CC-KV=V1&HW-CC-Date=20260312T211522Z&HW-CC-Expire=86400&HW-CC-Sign=F555E922E64B967F24F63F6A80C7BD1600265D0D29EC77D72B5014A06AFA454A)
    
-   方式二：
    1.  在函数列表中点击已创建的函数名称，进入函数详情页面。
        
        ![](https://contentcenter-vali-drcn.dbankcdn.cn/pvt_2/DeveloperAlliance_scene_100_1/ad/v3/MO_dqkOiTbSfMlFCaRxs8A/zh-cn_image_0000002474174529.png?HW-CC-KV=V1&HW-CC-Date=20260312T211522Z&HW-CC-Expire=86400&HW-CC-Sign=C315B5950ABCAE76A84DFFF50D5429CD61C08B473DA79CBF30ECFC6FFD1D72A5)
        
    2.  选择“函数代码”页签，点击“测试函数”。
        
        ![](https://contentcenter-vali-drcn.dbankcdn.cn/pvt_2/DeveloperAlliance_scene_100_1/f7/v3/ojba44gBSnOBGGdO00kLOw/zh-cn_image_0000002474214297.png?HW-CC-KV=V1&HW-CC-Date=20260312T211522Z&HW-CC-Expire=86400&HW-CC-Sign=00DB7CA333668A7291CEBE8B4EDC7AF38C6F11F6007E90985FC67B3E080FA5AC)
        
    3.  在右侧弹出的“测试函数”界面，使用默认测试事件、创建新测试事件或者使用已保存测试事件进行测试。
        -   使用默认测试事件
            
            直接点击“测试”对函数进行测试。
            
            ![](https://contentcenter-vali-drcn.dbankcdn.cn/pvt_2/DeveloperAlliance_scene_100_1/cd/v3/Kubvhu9mQv2JAop24-keyg/zh-cn_image_0000002440774524.png?HW-CC-KV=V1&HW-CC-Date=20260312T211522Z&HW-CC-Expire=86400&HW-CC-Sign=0ED54A2EE210B14CA466BB5238FB80C2E7FE29E4A3D6DCA68F8775C6F0B886BB)
            
        -   创建新测试事件
            
            如果需要设置调用函数的请求消息体，可按照如下步骤配置测试参数，并可保存为测试事件方便后续继续使用。
            
            1.  在“事件”文本框中输入JSON格式的事件参数，点击“保存”。然后在“提示”弹出框中输入事件名称，配置完成后点击弹出框右下角的“确认”。
                
                说明
                
                “事件”文本框内输入的JSON对象，对应的是触发器的event事件格式，会透传给函数。
                
                ![](https://contentcenter-vali-drcn.dbankcdn.cn/pvt_2/DeveloperAlliance_scene_100_1/3c/v3/PC2Wk0jqRa2kdu5zx7RvzQ/zh-cn_image_0000002440934320.png?HW-CC-KV=V1&HW-CC-Date=20260312T211522Z&HW-CC-Expire=86400&HW-CC-Sign=7FBF7A3E7E8C60253A9C119C4F1B695C7FA742B496DE7A49FF5E14FD2264AA4F)
                
            2.  点击“测试”，函数处理事件并返回测试结果。
        -   使用已保存测试事件
            1.  在“测试函数”界面，点击![](https://contentcenter-vali-drcn.dbankcdn.cn/pvt_2/DeveloperAlliance_scene_100_1/60/v3/O99Z8tLUQR6tfDlGzIK3Fg/zh-cn_image_0000002474174485.png?HW-CC-KV=V1&HW-CC-Date=20260312T211522Z&HW-CC-Expire=86400&HW-CC-Sign=1AC55241C6A4FEA33B26A28768278476456E647F59B54936F699453ECB72C7CB)展开已保存的测试事件列表，选择已配置的事件名称右侧的“加载”，然后点击“测试”，函数处理事件并返回测试结果。
                
                ![](https://contentcenter-vali-drcn.dbankcdn.cn/pvt_2/DeveloperAlliance_scene_100_1/4c/v3/bvHM9-kLQ261mtqYYingWA/zh-cn_image_0000002474174481.png?HW-CC-KV=V1&HW-CC-Date=20260312T211522Z&HW-CC-Expire=86400&HW-CC-Sign=4DF0C8CB922B2ECA49435731A62DF68CF0A2B2DE595A69C76929977FD6D4DB95)
                
            2.  （可选）如果需要删除已添加的测试事件，可在测试事件列表中点击事件名称右侧的“删除”即可删除测试事件。
                
                ![](https://contentcenter-vali-drcn.dbankcdn.cn/pvt_2/DeveloperAlliance_scene_100_1/b0/v3/PLwchSIVQWyWfK0vSihfDQ/zh-cn_image_0000002440934348.png?HW-CC-KV=V1&HW-CC-Date=20260312T211522Z&HW-CC-Expire=86400&HW-CC-Sign=CF6A60EE36F77CA3B2336FEB591C02C5152D957CF2DDB34BB462F86B055587DE)
                
    4.  查看测试结果。
        -   执行结果：展示测试后获得的响应结果。
            
            ![](https://contentcenter-vali-drcn.dbankcdn.cn/pvt_2/DeveloperAlliance_scene_100_1/90/v3/dO5yHj0XQ9aeR1voYa5XYA/zh-cn_image_0000002440774520.png?HW-CC-KV=V1&HW-CC-Date=20260312T211522Z&HW-CC-Expire=86400&HW-CC-Sign=B8A509F463EAFD325E4F7999F86760DAB0A77DEE27F05DDA333E838F75E40011)
            
        -   运行日志：展示函数运行过程中，通过logger API打印的日志，支持输出debug级别及以上日志（以下仅为日志级别测试结果）。
            
            ![](https://contentcenter-vali-drcn.dbankcdn.cn/pvt_2/DeveloperAlliance_scene_100_1/84/v3/hbD3uN91QBiH9TFhdHpskA/zh-cn_image_0000002474174461.png?HW-CC-KV=V1&HW-CC-Date=20260312T211522Z&HW-CC-Expire=86400&HW-CC-Sign=3853C599EBD88A81DC12E09175E754A9F5898DF2D9718E9AD0FBF2E1CB43FE3A)
            
        -   执行摘要：展示该次测试请求相关信息。
            
            -   请求ID：该条测试请求的RequestID，在后台日志中体现为X-Trace-ID。
            -   持续时间：函数执行的端到端时间。
            -   执行版本：该次调用测试的具体函数版本。
            
            ![](https://contentcenter-vali-drcn.dbankcdn.cn/pvt_2/DeveloperAlliance_scene_100_1/97/v3/-3qbOicEQ3KWn8IdD4886w/zh-cn_image_0000002474214337.png?HW-CC-KV=V1&HW-CC-Date=20260312T211522Z&HW-CC-Expire=86400&HW-CC-Sign=F96F1046491B865E1AAB4F0B16CDD4E35D525F535FB82EADE10D0EADF5D59E92)
            
    5.  “代码输入类型”为“在线编辑”的函数，测试过程中，如果需要修改函数入口文件代码，可直接在“函数代码”页签的代码编辑器中修改，然后点击页面底部的“提交”。当界面提示更新函数成功时，则可以点击“测试函数”对更改后的代码进行测试。
        
        ![](https://contentcenter-vali-drcn.dbankcdn.cn/pvt_2/DeveloperAlliance_scene_100_1/3e/v3/JMx73yWDREG-9De0EVEhkQ/zh-cn_image_0000002440774468.png?HW-CC-KV=V1&HW-CC-Date=20260312T211522Z&HW-CC-Expire=86400&HW-CC-Sign=5137374F5EB185726B227A3149B453C205CEA9022145CB68A5D4C408914E2AB0)
        
        “代码输入类型”为“.zip文件”的函数，测试过程中，如果需要修改函数代码文件，可在本地修改且打包完成后，点击![](https://contentcenter-vali-drcn.dbankcdn.cn/pvt_2/DeveloperAlliance_scene_100_1/54/v3/3VORz7woS6GTX8f53mcM1g/zh-cn_image_0000002474214305.png?HW-CC-KV=V1&HW-CC-Date=20260312T211522Z&HW-CC-Expire=86400&HW-CC-Sign=151D55E9E9759F5F81968D12C232A5139E1D5BCE6A34D7D2E77477FA87877FF2)重新上传函数部署包，然后点击页面底部的“提交”。当界面提示更新函数成功时，则可以点击“测试函数”对更改后的代码进行测试。
        
        说明
        
        如果代码更新量比较大，需要调整函数内存配置，可点击“内存配置”下拉框进行调整，然后再上传函数部署包。
        
        ![](https://contentcenter-vali-drcn.dbankcdn.cn/pvt_2/DeveloperAlliance_scene_100_1/74/v3/DDVWH7qRSbmDx_Bj9bUiVg/zh-cn_image_0000002440934324.png?HW-CC-KV=V1&HW-CC-Date=20260312T211522Z&HW-CC-Expire=86400&HW-CC-Sign=C8715C97452E3D5CE8BD5B011AB76E17ADD4DDB485F9E342C8A342742463DBB5)
        
    6.  函数测试无误后，可在“函数代码”页签点击“导出函数”导出函数部署包。导出包以“函数名称+函数版本.zip”格式命名，可查看函数结构和文件内容。
