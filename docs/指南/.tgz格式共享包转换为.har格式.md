# .tgz格式共享包转换为.har格式

使用DevEco Studio 3.1 Release之前创建出的、使用npm包管理器的Hvigor工程，构建出的共享包为.tgz格式的HAR包。要将已有的.tgz格式的HAR包转换成使用ohpm包管理器的.har格式的HAR包，需要根据以下的流程手动进行转换：

1.  将已有的.tgz格式的HAR包，使用可以解压.tgz格式的打包压缩工具进行解压。例如，在Windows 10/11或macOS上，可以使用操作系统自带的命令行工具tar：
    
    ```cangjie
    tar -xvf <tar package path> -C <extract path>
    ```
    
    ![](https://contentcenter-vali-drcn.dbankcdn.cn/pvt_2/DeveloperAlliance_scene_100_1/5d/v3/aAmJo6_PQyiCy8eo8U7yCA/zh-cn_image_0000002500909928.png?HW-CC-KV=V1&HW-CC-Date=20260313T050448Z&HW-CC-Expire=86400&HW-CC-Sign=7E62E202465CEA8D117544A635D3AEDD0F2813E653140C2E70B38C4892C76D3E)
    
    解压出来的文件会在解压路径的package文件夹中。
    
2.  将package文件夹中的package.json文件，根据[修改指导](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides-V14/ide-ohpm-historical-project-migration-V14#section62481731185412)，修改为oh-package.json5文件。
3.  使用步骤1的打包压缩工具，将package文件夹打包压缩为.har格式：
    
    ```lua
    tar -cvzf <tar package path> package
    ```
    
    ![](https://contentcenter-vali-drcn.dbankcdn.cn/pvt_2/DeveloperAlliance_scene_100_1/12/v3/OzgSqYQDQEWrwZAbSs4EuQ/zh-cn_image_0000002532669841.png?HW-CC-KV=V1&HW-CC-Date=20260313T050448Z&HW-CC-Expire=86400&HW-CC-Sign=9DDA8AB681C0AC8F49BCEC3E40D12B7ABCC9B4B20C0E112903946186FE77040C)
    

转换后.har格式的HAR包可以使用ohpm包管理器进行安装与发布。
