# 签名证书文件解析错误

**问题现象**

打包签名提示“**DerValue.getOID, not an OID 49 Detail: Please check the message from tools**”错误。

**可能原因**

解析证书文件失败，一般情况是由于用户传入了非标准证书文件或证书文件损坏而导致。

**常见错误场景**

Certpath file配置了错误的证书文件。

![](https://contentcenter-vali-drcn.dbankcdn.cn/pvt_2/DeveloperAlliance_scene_100_1/86/v3/EdAk0dZzT9OTKqDL693Q7g/zh-cn_image_0000002342518434.png?HW-CC-KV=V1&HW-CC-Date=20260313T054926Z&HW-CC-Expire=86400&HW-CC-Sign=B0A81368A9DEBB40389B6CA07B18675EE4370FCD0003EE868FA7F6B539A3C82A)

**解决措施**

检查Certpath file配置的证书文件是否为标准证书文件，检查方式如下：

DevEco Studio Terminal窗口使用keytool命令查看配置的证书文件，示例：keytool -printcert -file ${Certpath file}。

-   格式正确的证书文件。
