# 调试过程中无法添加断点

**问题现象**

调试过程中无法添加断点，提示“Can't set breakpoint to remote debug server”，如图所示。

![](https://contentcenter-vali-drcn.dbankcdn.cn/pvt_2/DeveloperAlliance_scene_100_1/72/v3/hDP9se64TJW_QJaxCfvQ0A/zh-cn_image_0000002250578541.png?HW-CC-KV=V1&HW-CC-Date=20260313T055456Z&HW-CC-Expire=86400&HW-CC-Sign=587051B59160B54F65F812EB62495EA9F97EA3D642082F7D63A7250B38A12CB3)

**解决措施**

请使用以下方法排查原因：

1.  检查是否存在xx.map文件。如果不存在，则需重新编译构建生成map文件，然后进行断点调试。
    
    -   对于stage模型工程，需检查sourceMaps.map文件是否存在。该文件由编译构建生成，位于模块的“entry\\build\\default\\intermediates\\loader\_out\\default”目录下，如下图所示：
        
        **图1** stage模型工程中sourceMaps.map文件目录  
        ![](https://contentcenter-vali-drcn.dbankcdn.cn/pvt_2/DeveloperAlliance_scene_100_1/5e/v3/KTCsNJ6ZS0WxMZxh1RIq0Q/zh-cn_image_0000002250498437.png?HW-CC-KV=V1&HW-CC-Date=20260313T055456Z&HW-CC-Expire=86400&HW-CC-Sign=E6E3269FD711BA2A3A427FFF3F6F264E727C47D0B4265E2AFE71DA7AF2417367)
        
    -   对于FA模型工程，需检查断点所在文件对应的map文件是否存在。该文件由编译构建生成，位于模块的“entry\\build\\default\\intermediates\\loader\_out\\default”目录下，如下图所示：
        
        **图2** FA模型工程中map文件目录  
        ![](https://contentcenter-vali-drcn.dbankcdn.cn/pvt_2/DeveloperAlliance_scene_100_1/b5/v3/ED6yLvj0RF-DAWhHKlLhrw/zh-cn_image_0000002215658548.png?HW-CC-KV=V1&HW-CC-Date=20260313T055456Z&HW-CC-Expire=86400&HW-CC-Sign=937E9087644FCF4435228F866E2E8E1F970D27655BF74DF7ECACCDCB22437083)
        

2.  检查代码文件是否已加载。启动调试后，如果断点所在代码文件未加载（已加载的代码文件会显示在下方**Console**中），则断点将无法添加。程序运行并加载代码文件后，断点即可正常添加。代码文件未加载的原因是未被入口模块直接或间接引入。
    
    **图3** 断点所在代码文件未加载  
    ![](https://contentcenter-vali-drcn.dbankcdn.cn/pvt_2/DeveloperAlliance_scene_100_1/8c/v3/h2ExWGLEQD-f2O1FqDSKtQ/zh-cn_image_0000002215498752.png?HW-CC-KV=V1&HW-CC-Date=20260313T055456Z&HW-CC-Expire=86400&HW-CC-Sign=A4F358ECAE735A8795B98A3DC8FDCEF40914D859D26D8EF7CD6F7347B6B60841 "点击放大")
    
3.  断点添加位置无效。ArkTS不支持在方法的右括号单独所在行添加断点。
    
    **图4** 断点位置无效  
    ![](https://contentcenter-vali-drcn.dbankcdn.cn/pvt_2/DeveloperAlliance_scene_100_1/a0/v3/FDqgnbJrTNWuE6gYZzXtxQ/zh-cn_image_0000002250578549.png?HW-CC-KV=V1&HW-CC-Date=20260313T055456Z&HW-CC-Expire=86400&HW-CC-Sign=1A00086E1DB7B3A74CAD2403D9E13FE77F50CE5BF2C7E371F0899452E643DD06 "点击放大")
